Variables

1. Connect to Postgres

Connect as the admin user:

bash
Explain

Opens the PostgreSQL interactive terminal (psql) as . With no host given, psql connects over a local Unix socket. Database defaults to one matching the username.

Flags used

-U, --username=USERNAME
Database user name.

Other flags you might want

-d, --dbname=DBNAME
Connect to a specific database instead of the user-named default.
-l, --list
List available databases, then exit (no interactive session).
-w, --no-password
Never prompt for a password (useful in scripts where prompts hang).
-W, --password
Force a password prompt even if psql would otherwise skip it.
-X, --no-psqlrc
Skip ~/.psqlrc — clean session without user customisations.

Or against a specific host/port:

bash
Explain

Connects to a TCP host instead of the default Unix socket — required when Postgres runs in Docker, on another machine, or on a non-standard port.

Flags used

-h, --host=HOSTNAME
Database server host or socket directory.
-p, --port=PORT
Database server port (Postgres default is 5432).
-U, --username=USERNAME
Database user name.

Other flags you might want

-c, --command=COMMAND
Run a single SQL command and exit — perfect for scripting.
-f, --file=FILENAME
Execute commands from a file, then exit.
-v, --set=NAME=VALUE
Set a psql variable, e.g. -v ON_ERROR_STOP=1 to abort on first error.
-1, --single-transaction
Wrap the whole non-interactive session in one transaction.
--csv
Switch output to CSV — handy when piping query results elsewhere.

Using a connection string:

bash
Explain

Passes connection info as a single URI. Useful when you already have a DATABASE_URL from a cloud provider or .env file. The path component () is the database name to connect to.

URI parts

postgres://
Scheme — postgres:// and postgresql:// are interchangeable.
user:password@
Credentials (URL-encode special characters).
host:port
Server location.
/dbname
Database to connect to. Omit to connect to the user-named default.
?param=value
Optional query params like sslmode=require, application_name=…, connect_timeout=10.

Common query params

sslmode=require
Force TLS (also: verify-ca, verify-full).
connect_timeout=10
Fail fast if the server isn't reachable.
application_name=my_app
Tag your session for easier debugging via pg_stat_activity.

3. Connect to the new database

Inside psql:

Explain

The \c (or \connect) backslash command reconnects the current psql session to a different database — no need to quit and start a fresh process.

Variations

\conninfo

Constraints used

BIGSERIAL
Shorthand for BIGINT NOT NULL DEFAULT nextval('seq') — 64-bit auto-increment column. For Postgres 10+, prefer GENERATED ALWAYS AS IDENTITY.
PRIMARY KEY
Implies UNIQUE + NOT NULL and creates a btree index.
NOT NULL
Disallows NULLs at insert time.
UNIQUE
Adds a unique constraint (and supporting index).
DEFAULT
Server-side default if the column is omitted in an INSERT.
TIMESTAMPTZ
Timestamp with timezone — almost always preferable to plain TIMESTAMP.

Other clauses you might want

CHECK (expr)
Arbitrary row-level validation, e.g. CHECK (length(email) < 320).
REFERENCES other(id)
Inline foreign key.
GENERATED ALWAYS AS IDENTITY
Modern replacement for SERIAL / BIGSERIAL (SQL-standard).
IF NOT EXISTS
Idempotent variant of CREATE TABLE.
PARTITION BY RANGE/LIST (col)
Declarative partitioning for large tables.

6. Verify

psql
Explain

psql backslash commands for introspection. They don't return rows like a SELECT — psql generates the matching catalog query for you and pretty-prints the result.

Commands used

\l
List databases on the server.
\dn
List schemas in the current database.
\dt pattern
List tables matching a pattern (schema-qualified glob).
\d name
Describe a table/view/sequence — columns, indexes, constraints.

Other introspection commands

\du
List roles (users + groups).
\dp
Show table-level privileges.
\df
List functions.
\di
List indexes.
\d+ name
Verbose describe — includes storage info, comments.
\?
Show all backslash commands.

7. Tear down (optional)

sql
Explain

Reverses the setup. Order matters: drop dependent objects (table) before their container (schema, database). DROP DATABASE cannot run while any session is connected to that database — disconnect first or use the optional clause below.

Clauses used

CASCADE
Also drop everything that depends on this object (tables in the schema, foreign keys, views, etc.).

Other clauses you might want

IF EXISTS
Don't error if the object is already gone — idempotent teardown.
RESTRICT (default)
Refuse to drop if dependents exist.
DROP DATABASE … WITH (FORCE)
Postgres 13+ — terminate connected sessions, then drop.

One-shot version (no interactive psql)

bash
Explain

Two-step setup that needs no interactive psql session. The heredoc (<<'SQL' … SQL) feeds the statements to psql on stdin. The single quotes around the delimiter ('SQL') disable shell variable expansion inside the body, so $ in your SQL is left alone.

Useful additions

-v ON_ERROR_STOP=1
Make psql exit non-zero on the first SQL error — essential in scripts.
-1 (single-transaction)
Wrap the whole heredoc in one transaction so a mid-script failure rolls everything back.
-e / -a
Echo each statement (or all input) for CI logs.
