CRF (Constant Rate Factor) is the recommended quality knob for libx264/libx265. Lower CRF = higher quality + larger file. Useful range is 18–28; 23 is the libx264 default. The preset trades encoding speed against compression efficiency.

-preset NAME
Encoding speed/compression tradeoff: ultrafast, superfast, veryfast, faster, fast, medium, slow, slower, veryslow.
-c:v CODEC
Video codec — typically libx264 or libx265 for CRF mode.

Audio codec.

If the existing codecs are legal in the target container, ffmpeg can re-mux instantly — no quality loss, no CPU spent decoding/encoding. Common moves: .mkv → .mp4 (H.264 + AAC), .mov → .mp4, .ts → .mp4.

bash
Explain

Re-muxes into a different container () without touching the encoded streams. If the codecs don't fit the new container, ffmpeg will error out — switch from -c copy to an explicit re-encode in that case.

Flags used

-i FILE
Input file.
-c copy
Stream-copy every stream (video, audio, subtitles). Shorthand for -c:v copy -c:a copy -c:s copy.

Other flags you might want

-map 0
Copy all streams from the input, including chapters / data tracks.
-bsf:a aac_adtstoasc
Bitstream filter needed when copying AAC from MPEG-TS into MP4.
-movflags +faststart
Re-locate moov atom to the start of the file — required for progressive HTTP playback.
-fflags +genpts

-f FORMAT
Force the output muxer when the extension is ambiguous (e.g. -f mp4).

Weight palette to moving (diff) regions — best for talking-head video.
