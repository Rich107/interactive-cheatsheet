12. Multiple inputs and slurp

-s (slurp) reads all inputs into a single array — combine multiple JSON documents into one:

bash
Explain

Without -s, jq applies the filter once per input value (one per file, or one per top-level value in NDJSON). With -s, all inputs are read first and packed into a single array, which becomes the input to the filter.

Flags used

-s, --slurp
Read all inputs into a single array; the filter sees that array as its sole input.

Related flags / builtins

-n, --null-input
Filter starts with null; use inputs to consume the stream explicitly.
inputs
(With -n) generator yielding each remaining input value.
input
Pull a single additional input value.
add
Merge / concatenate / sum across the slurped array.
-R, --raw-input
Treat each input line as a string instead of JSON; with -s you get the whole file as one string.
