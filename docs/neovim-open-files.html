neovim — opening files & diffing revisions

Every useful way to open files in Neovim — including diffing across git revisions. Change the variables below; every snippet updates live. Shell snippets use bash; in-nvim ex commands use vim. Flag text captured from nvim --help on Neovim 0.11.6; :Gvdiffsplit-family snippets require the vim-fugitive plugin.

Variables

Other flags you might want

"Factory defaults" — skip user config, plugins, and shada. The fastest way to confirm a bug is config-related.

Explain

Opens a vertical split containing the version of the current file at and enters diff mode. Bare :Gvdiffsplit (no argument) compares against the index/HEAD.

Requires the vim-fugitive plugin. Install via your plugin manager, e.g. Plug 'tpope/vim-fugitive' / {'tpope/vim-fugitive'} in lazy.nvim. Without fugitive, :Gvdiffsplit errors with E492: Not an editor command; see the fallback snippet below.

Arguments used

{rev}
Any git revision — commit SHA, branch, tag, HEAD~N, main, origin/main, etc.

Related fugitive commands

:Gvdiffsplit
No-arg form — vertical diff against the index.
:Gvdiffsplit!
During a merge conflict — diff against both parents in two splits.
:Gdiffsplit
Same idea, horizontal split.
:Gtabedit {rev}:%
Open the revision of the current file in a new tab — handy for read-only browsing.
:Git blame
Inline blame for the current buffer with quick-jump bindings.
:Gread {rev}:%
Replace the current buffer with the revision's contents (no diff) — useful for "give me the old version back".

Plain vim alternative (no fugitive) — read git show output into a scratch buffer and diff both windows manually:

vim
Explain

Step by step: open a new vertical window, read git show REV:./FILE into it, delete the empty first line that :r leaves behind (0d_), mark the buffer as a scratch (no file, no swap, no buflisted), then enable diff. Switch back to the original window (:wincmd p) and enable diff there too. Works on a stock Neovim with no plugins.

Pieces used

:vert new
Open a new empty vertical-split window.
:r !cmd
Read the output of a shell command into the buffer below the cursor.
0d_
Delete line 1 (the blank line :r leaves) into the black-hole register.
setlocal buftype=nofile nobuflisted noswapfile
Mark as a scratch buffer — :q won't prompt and there's no .swp.
:diffthis
Enable diff in the window.
:wincmd p
Switch focus to the previous window.

--global vs --local

12. Open at the precise location of a git conflict

Jump straight to the first conflict marker on open — escape the angle brackets so the shell doesn't try to redirect:

bash
Explain

Opens and jumps to the first line starting with <<<<<<< — the start marker git inserts for an unresolved conflict. Use n in normal mode to jump to the next conflict.

The single-quotes around '^<<<<<<< ' stop the shell interpreting < as input redirection. The ^ anchors the search to start-of-line so it only matches actual conflict markers, not text that happens to contain seven angle brackets.

Flags used

+/PATTERN
Run a forward search after the file loads. See section 1.

Related patterns

+/'^======='
Jump to the divider between the two conflicting versions.
+/'^>>>>>>> '
Jump to the end marker of a conflict.
+/'^|||||||'
Conflict base marker (only present with merge.conflictStyle=diff3).

Combine with -O to open every conflicted file side-by-side, each starting at its first conflict — one shell line for the whole rebase:

bash
Explain

git diff --name-only --diff-filter=U prints just the unmerged ("Unmerged") paths. Shell expansion turns them into nvim args. -O opens each in its own vertical split; the +/ search runs in the first one and lands the cursor on the first conflict.

Flags used

-O
Vertical splits, one per file.
+/PATTERN
Initial search.
git diff --diff-filter=U
Restrict output to files in the "Unmerged" state.
git diff --name-only
Print just filenames, no diff content.
If many files are conflicted you'll want -p (one tab per file) instead of -O — too many vertical splits become unreadable.
