tmux quickref

Change the variables below — every snippet on the page updates live. Most tmux usage is in-tmux keybindings prefixed by Ctrl-b (rebind via the prefix key input). Click Explain under any block for flag/key details and related options.

1. Start & attach sessions

Create a named session detached from your current terminal, naming the first window and launching a command inside it:

Explain

Creates session with its first window named running . Without -d, tmux attaches immediately; add -d to script setup without jumping into the session.

Flags used

-s session-name
Name the new session. Names must be unique on the server.
-n window-name
Name the first window created with the session.
shell-command
Command to run in the new window (positional, after flags). Defaults to your login shell.

Other flags you might want

-d
Create detached — don't attach the current client. Essential for scripting.
-A
Attach to an existing session of the same name instead of erroring.
-D
With -A, detach any other clients already attached.
-c start-directory
Working directory for the first window's process.
-x width -y height
Force initial size when starting detached (otherwise tmux uses an 80x24 default until a client attaches).
-e KEY=VALUE
Set an environment variable in the new session (repeatable).

Explain

Explain

-a
Kill all sessions except the target — useful for cleanup.

Clear alerts (bell, activity, silence) for the session instead of killing it.

Nuclear option — kill every session and stop the tmux server process itself:

bash
Explain

Stops the tmux server, terminating all sessions, windows, panes, and the processes running inside them. There's no confirmation — use sparingly.

tmux kill-pane -t SESSION:WINDOW.PANE

For precise sizing, open the tmux command prompt with {{prefix_key}} : and resize by an exact number of cells:

tmux
Explain

Run from the tmux command prompt (:). The directional flags resize by an adjustment value (default 1); -x/-y set absolute dimensions.

Flags used

-L / -R / -U / -D
Direction to grow or shrink.
adjustment
Trailing positional number — how many cells to resize by.
-x width
Set absolute width in columns.
-y height
Set absolute height in rows.

Other flags you might want

-Z
Toggle zoom on the target pane.
-t target-pane
Pane to resize (default: active pane).
-T
Trim lines below the cursor (rarely needed).
-M
Begin mouse resize drag (used by mouse bindings).

Explain

Copy mode replaces your shell prompt with a navigable view of the pane's scrollback buffer. The keybindings shown are the default emacs mode; with setw -g mode-keys vi in your config you get vi-style motions (h/j/k/l, v to start selection, y to yank).

Beginning / end of the current line.
Ctrl-u / Ctrl-d

Paste a buffer into a specific pane.

8. Send keys to a pane from the shell

Send a command line to a specific pane without attaching to its session — note the literal Enter token so the shell inside the pane actually executes it:

bash
Explain

Types echo hello followed by a newline into pane of window in session . Each positional argument is either a literal string or a tmux key name (Enter, C-c, Escape, BTab, etc.).

Flags used

-t target-pane
Target in session:window.pane form. Window can be name or index; pane is its index.
key ...
Positional list of strings and/or key names to send.

Other flags you might want

-l
Send keys literally — don't interpret Enter/C-c as key names.
-R
Reset the terminal state of the target pane before sending.
-N repeat-count
Repeat the key sequence N times.
-H
Interpret arguments as hex bytes (e.g. 0x0d for Enter).
-M
Send a mouse event (used internally by mouse bindings).

To send a Ctrl-C to interrupt a running process: tmux send-keys -t {{session_name}}:{{window_name}}.{{target_pane}} C-c.

Size of the new pane — a number of cells or a percentage like 30%.

-P

-d

10. Common ~/.tmux.conf tweaks

A minimal, opinionated starting config. Drop it in ~/.tmux.conf and reload with tmux source-file ~/.tmux.conf (or use the {{prefix_key}} r binding included at the bottom):

tmux.conf
Explain

Each line is a tmux command run at startup. Edit and re-source freely — these are safe to tweak and don't require restarting the server.

Directives used

set -g OPTION VALUE
Set a global server/session option (-g = global).
setw -g OPTION VALUE
Shorthand for set-window-option — options that apply per window.
set -ga OPTION VALUE
-a appends to a list-valued option instead of replacing it.
unbind KEY
Remove an existing binding.
bind KEY COMMAND
Bind a key (under the prefix table) to a tmux command.
send-prefix
Pass the prefix through to the running program — needed so apps still see Ctrl-a.

Other options you might want

set -g status-position top
Move the status bar to the top of the screen.
set -g escape-time 10
Reduce the delay after Escape (helps Vim/Neovim feel snappier).
set -g focus-events on
Forward terminal focus events to running programs.
setw -g monitor-activity on
Highlight windows with new output.
set -g aggressive-resize on
Resize windows based on the smallest attached client showing them.
set -g status-interval 5
How often the status line refreshes (seconds).
