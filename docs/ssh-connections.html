2. Generate a key pair

Modern Ed25519 — small, fast, secure. Press Enter at the passphrase prompt to leave it empty, or pick a passphrase for anything that lives long-term on disk:

bash
Explain

Generates an Ed25519 key pair at (private) and .pub (public), stamping the public key with the comment so you can recognise it later.

Flags used

-t TYPE
Key algorithm: ed25519, rsa, ecdsa, ed25519-sk / ecdsa-sk (hardware-backed).
-C COMMENT
Comment string appended to the public key — usually user@host so you can tell keys apart in authorized_keys.
-f FILE
Output path for the private key. .pub is appended for the public half.

Other flags you might want

-N PASSPHRASE
Provide the passphrase on the command line (e.g. -N "" for an empty one — non-interactive).
-a ROUNDS
KDF rounds for the encrypted private key (higher = slower to brute-force; default 16).
-q
Quiet — suppress the "Generating…" banner.
-o
Use the new OpenSSH key format (default for ed25519).

Legacy RSA for hosts that don't accept Ed25519 yet (older network gear, some CI runners). Always pick at least 4096 bits:

Explain

Generates a 4096-bit RSA key pair at ~/.ssh/id_rsa with the comment .

Flags used

-t rsa
RSA algorithm.
-b BITS
Key length. 4096 is the modern minimum; 2048 is now considered weak.
-C COMMENT
Identifying comment on the public key.
-f FILE
Output filename.

Other flags you might want

-t ecdsa -b 521
NIST P-521 ECDSA — smaller than RSA, broader support than ed25519 on some BMC / appliance firmware.
-t ed25519-sk
Ed25519 key bound to a FIDO2 security key (YubiKey etc.).
-Z CIPHER
Cipher used to encrypt the private key on disk.

Explain

While the ssh connection is up, anything that connects to localhost: on your laptop is tunnelled to : as seen from . The forward shuts down when the ssh session ends.

Flags used

-L [BIND:]PORT:HOST:HPORT
Forward local PORT to HOST:HPORT through the ssh connection. Without a bind address it listens on 127.0.0.1; use *:PORT:… to bind all interfaces.

Other flags you might want

-N
Don't execute a remote command — pure tunnel mode.

Fork to background once authentication succeeds.

Abort the connection if the forward can't be set up — avoid silently broken tunnels.

11. SSH config file

Drop a block like this into ~/.ssh/config and ssh app is enough — every flag becomes a default for that host alias:

sshconfig
Explain

Defines a host alias app. All subsequent ssh app, scp file app:/path, and rsync app:/path . calls inherit these defaults. Per-host blocks are matched top-to-bottom; the first match for each option wins.

Directives used

Host PATTERN
Alias / glob pattern this block applies to.
HostName
Real DNS name or IP to connect to (lets Host stay a short nickname).
User
Login user.
Port
Remote sshd port.
IdentityFile
Private key to offer. Multiple lines allowed.
ProxyJump
Bastion(s) to hop through — same effect as ssh -J.
ServerAliveInterval
Send a keepalive probe every N seconds of idle time.
ServerAliveCountMax
Disconnect after this many missed keepalives.
ForwardAgent
Whether to forward your ssh-agent socket onto the remote (default no — keep it that way unless required).

Other directives you might want

IdentitiesOnly yes
Only offer keys listed in this block — stops "too many auth failures" on multi-key hosts.
UseKeychain yes
(macOS) Save / fetch the passphrase from the login keychain.
AddKeysToAgent yes
Add the unlocked key to the agent on first use.
StrictHostKeyChecking accept-new
Auto-trust first-seen host keys, still reject changes.
ControlMaster auto   ControlPath …   ControlPersist 10m
Multiplex multiple ssh sessions over one TCP connection — huge speed-up for tools that fire many ssh commands.
LocalForward / RemoteForward / DynamicForward
Config-file equivalents of -L / -R / -D.
